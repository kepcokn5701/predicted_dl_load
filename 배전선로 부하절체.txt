너는 KEPCO(한국전력공사)의 송변전 및 배전 데이터 분석 전문가이자 파이썬 개발자야. 
기존의 XGBoost 부하예측 로직을 '1년 전 동일 주차 데이터 동기화(Yearly-Week-Sync)' 방식으로 고도화해줘.

### 1. 배경 및 목적
- 한 달 평균 데이터는 해상도가 낮아 요일별(평일/주말) 특성을 반영하지 못함.
- 실무자가 휴전 일자를 판단할 때 가장 신뢰하는 '작년 이맘때 동일 주차'의 패턴을 추출해야 함.

### 2. 핵심 로직 (v5.5)
- [Target Date]: 사용자가 입력한 휴전 예정일.
- [Reference Week]: Target Date로부터 정확히 364일(52주) 전의 날짜를 기준으로, 해당 날짜가 포함된 월요일~일요일(00:00~23:00) 데이터를 추출.
  * 364일을 사용하는 이유는 요일(Day of Week)을 완벽히 일치시키기 위함임.
- [Feature Engineering]: 
  1. 추출된 1년 전 주간 데이터를 시간대별(24시간) 평균 패턴으로 가공.
  2. 이를 현재 XGBoost 모델의 입력 피처(last_year_same_week_load)로 결합.
  3. 최근의 전반적인 전력수요 증감 트렌드(Scaling Factor)를 반영하여 최종 보정.

### 3. 기능 요구사항
1. 데이터 전처리: 'datetime' 컬럼을 기준으로 1년 전 동일 주 데이터를 슬라이싱하는 함수 구현.
2. 부하 절체 시뮬레이션: 예측된 부하값에 1/N 절체 로직을 적용하고, 임계치(14.0MW) 가부 판정.
3. 분석 로그 출력: 실무자가 신뢰할 수 있도록 아래 예시와 같은 정교한 로그를 출력할 것.

[로그 예시]
- 분석 모드: Yearly-Week-Sync v5.5
- 타겟 일자: 2026-01-21 (수)
- 대조 구간: 2025-01-20 (월) ~ 2025-01-26 (일)
- 1년 전 동일 요일 평균 부하: [값] MW
- 최종 예측 결과 및 가부 판정: [SUCCESS/FAIL]

### 4. 출력 형식
- 주석이 상세히 달린 Python 코드를 작성해줘.
- 데이터프레임 구조는 'datetime', 'load' 컬럼을 기본으로 가정해.

참조주차평균이 지금은 타겟날짜의 24시간의 평균부하로 돼있는데, 그게 아니고, 참조주차평균을   reference_week의 같은시간대의 평균으로 바꿔줘. 아래는 예시자료니까 참고해줘.

[15:17:29] │ ② Predicted_Load = Base_Load × Trend_Factor                    │
[15:17:29] │    → Trend_Factor = (Target요일부하) / '(참조주차평균)'<-수정필요            │

---------

주말때문에 팩터의 계수값이 너무 크게 나옴, 수정필요. 예측하고자하는 날의 364일을 뺀 날짜의 그 일주일의 주말을 뺀 평일데이터만 계산. 
[15:55:00] │ ② Predicted_Load = Base_Load × Trend_Factor[hour]              │
[15:55:00] │    → Trend_Factor[hour] = (Target요일 hour시) / (참조주차 hour시)│
[15:55:00] │    → 같은 시간대끼리 비교하여 보정 계수 산출                     │
